NOTE: These files use flat-namespace imports (from SHMProtocol import ...) designed
for TouchDesigner's Python environment where each Text DAT in a component is an
importable module by its DAT name.  They cannot be imported as a regular Python
package — there is intentionally no __init__.py here.

LIBRARY MODE (v1.7.0+): With cuda_link installed via install_td_library.cmd (or
pip install cuda_link), the COMP only needs the 6 core TD-glue DATs plus callback
DATs.  The 14 mirror Text DATs (listed under files_mirror below) can be removed from
the .tox — CUDALinkBootstrap.py injects sys.path and aliases them from the pip
package.  Verify with: [CUDALinkBootstrap] Library mode active in the Textport.

CLASSIC/FALLBACK MODE: If no pip install is present, deploy ALL files (glue +
mirrors + callbacks) as Text DATs.  CUDAIPCExtension.py detects the absent bootstrap
and falls back to the DAT-based imports automatically.

To deploy into a .tox component: copy each file into a Text DAT whose name matches
the filename (without .py), e.g. CUDAIPCExtension.py -> Text DAT named CUDAIPCExtension.
CUDALinkBootstrap must be the FIRST DAT loaded (first in COMP load order).
Use the upstream TOXES/CUDAIPCLink_v1.7.2.tox as the reference component template.
